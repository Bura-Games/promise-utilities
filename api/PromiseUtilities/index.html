PromiseUtilities

Utilities for working with Promises.

Functions​

FastAsyncMulti​

</>
PromiseUtilities.FastAsyncMulti(callback: (Arguments...) → Returns...) → (Arguments...) → Promise<Returns...>

A function that turns another function into the same as an async function in ECMAScript. This supports multiple returns however. 

async function promiseUpperAndLower(value: string) {
	return [value.toUpperCase(), value.toLowerCase()];
}

would basically be equivalent to:

local function GetUpperAndLowerAsync(value: string)
	return string.upper(value), string.lower(value)
end

local PromiseUpperAndLower = PromiseUtilities.FastAsyncMulti(GetUpperAndLowerAsync)

Async​

</>
PromiseUtilities.Async(callback: (Arguments...) → Returns) → (Arguments...) → Promise<Returns>

A function that turns another function into the same as an async function in ECMAScript. This was taken from here. 

async function promiseGet(url: string) {
	return HttpService.GetAsync(url);
}

would basically be equivalent to:

local function GetAsync(url: string)
	return HttpService:GetAsync(url)
end

local PromiseGet = PromiseUtilities.Async(GetAsync)
INFO

This is meant to be entirely equivalent to the TypeScript version. I would recommend using PromiseUtilities.FastAsync instead for better performance.

FastAsync​

</>
PromiseUtilities.FastAsync(callback: (Arguments...) → Returns) → (Arguments...) → Promise<Returns>

A function that turns another function into the same as an async function in ECMAScript. This uses a thread recycling "fast spawn" function instead of coroutine.wrap. 

async function promiseGet(url: string) {
	return HttpService.GetAsync(url);
}

would basically be equivalent to:

local function GetAsync(url: string)
	return HttpService:GetAsync(url)
end

local PromiseGet = PromiseUtilities.FastAsync(GetAsync)

AwaitMulti​

This is a yielding function. When called, it will pause the Lua thread that called the function until a result is ready to be returned, without interrupting other scripts. Yields
</>
PromiseUtilities.AwaitMulti(...: T | Promise<T...>) → T

The functionality of this is meant to be the same as the await keyword in ECMAScript. This allows returning multiple values! 

local function PromiseUpperAndLower(value: string): Promise<string, string>
	return Promise.new(function(resolve)
		resolve(string.upper(value), string.lower(value))
	end)
end
local upper, lower = PromiseUtilities.AwaitMulti(PromiseUpperAndLower("Hello there!"))
Behavior

This function is almost entirely identical to Promise.expect. You can probably just use that instead!

Errors

TypeDescription
CancelledErrorThrown if the Promise was cancelled.
RejectedErrorThrown if the Promise was rejected.

Await​

This is a yielding function. When called, it will pause the Lua thread that called the function until a result is ready to be returned, without interrupting other scripts. Yields
</>
PromiseUtilities.Await(promiseOrValue: Promise<T> | T) → T

The functionality of this is meant to be the same as the await keyword in ECMAScript. 

async function promiseGet(url: string) {
	return HttpService.GetAsync(url);
}

const result = await promiseGet("https://example.com");

-- This is good practice - you don't really lose debug info this way.
local function GetAsync(url: string)
	return HttpService:GetAsync(url)
end

local PromiseGet = PromiseUtilities.FastAsync(GetAsync)
local result = PromiseUtilities.Await(PromiseGet("https://example.com"))
Behavior

This function is almost entirely identical to Promise.expect. You can probably just use that instead!

Errors

TypeDescription
CancelledErrorThrown if the Promise was cancelled.
RejectedErrorThrown if the Promise was rejected.
